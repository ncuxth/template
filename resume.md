## 项目经验
**一、 多实现消息推送系统**
**项目描述：**
一个为业务发送消息需求设计的**通用消息推送中间件**。支持邮件、短信、飞书等多种渠道的消息推送，并统一了对外接口协议,使用消息中转站削峰填谷,并提供了**Mysql**和**kafka**两种中转站的实现,优化**集群**部署实现高可用.
**主要工作：**
1. 使用**kafka**实现消息中转站,用来临时存储消息,并达到削峰填谷的作用,提高系统的吞吐量,支撑 **3000+ QPS** 的高并发吞吐,使用**Mysql**和**定时任务**实现消息中转站,避免引入kafka而降低系统可用性,避免增加系统复杂度
2. 实现**三优先级**通道,在消费者端控制三个通道的处理频率,确保高优先消息的优先消费,使用redis实现在滑动时间窗口内的**限额控制**,并在获取限额配置时使用redis作为**缓存**,保证系统性能
3. 使用redis的**Zset**和mysql实现**秒级**精度的**延时消息**,能够兼顾大数据量的存储和扫描的高性能需求
4. 项目支持**集群部署**以提高可用性,为此对消费者端做了优化以**避免消息重复消费**,使用**redisson分布式锁**配合看门狗机制,避免mysql实现下的重复消费和**锁过期问题**,将多个实例的消费者分配到同一**消费者组**,防止kafka实现的重复消费问题
5. 实现**多厂商容灾切换机制**，实时监测第三方平台调用状态，一旦发现持续异常（如阿里云短信故障）自动切换至备用服务商（如腾讯云），保证**高可用性**;

**二、自动化运维Agent**
**项目描述：**
一款大模型驱动的**自动化运维**辅助系统。通过整合企业内部知识库、智能对话与自动化运维链路，实现问题的**及时应答**与故障的**智能定位和排障**，能够降低团队**运维人力成本**，缩短故障响应时间。
**主要工作：**
1. 基于 **SpringAI**与**LangGraph**框架设计并实现**Plan-Execute Agent**,能够自动对系统的故障定位和排查,使用**React Agent**用于快速响应问题，提高大模型排查问题的**效率**,接入tool和**mcp**工具提高模型**能力边界**
2. 构建 **RAG 知识库系统**，优化文档分块策略,将RAG检索注册为**Tool**,使对知识库的查询更加灵活,使用**本地Cross-Encoder**或**Rerank大语言模型**双实现对检索结果**重排**,大大提高**复杂语义场景**的知识检索准确率
3. 实现基于 **ReAct架构** 的智能对话Agent，支持**多轮对话上下文记忆**与**上下文压缩机制**，在**保留上下文记忆**的情况下Token用量显著降低；采用SSE实现**流式输出**，降低首字响应延迟,优化用户体验；
4. 实现基于**Plan-Execute架构**的系统性自动运维，系统根据告警信息**自动规划步骤和调用工具分析**，并生成报告,能够快速定位问题并提出解决方案,子Agent使用**不同模型**实现,兼顾**计划质量和执行速度**。
5. 实现基于 Spring Security + JWT + **Redis黑名单**的认证鉴权机制，从而实现无状态认证下的**主动失效**能力。
